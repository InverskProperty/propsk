    1. Dual ID System Confusion

    The Problem:
    Your system has two types of IDs that were getting mixed up:

    - Database IDs (Primary Keys): Sequential integers like 24, 210, etc.
    - PayProp IDs (External API IDs): Strings like "7QZGYw4nX9", "dMQNzrzlKp"

    What Was Broken:
    - PropertyService was generating fake hash IDs (like 888678848) from PayProp IDs
    - URLs used these fake IDs, causing "Property not found" errors
    - Database lookups failed because fake IDs don't exist in the database

    How It Should Work:
    - Internal operations (database queries, relationships): Always use database IDs
    - External API calls (PayProp integration): Use PayProp IDs
    - URLs and navigation: Use database IDs for consistency
    - Property entities bridge both systems by storing both ID types

    2. Entity Architecture Mismatch

    The Problem:
    Your system has parallel entity hierarchies that were being used inconsistently:

    Customer (Main table) ‚Üê‚Üí PropertyOwner/Tenant/Contractor (Specialized tables)
         ‚Üì                           ‚Üì
    Used in list pages          Used in dashboard counts
         ‚Üì                           ‚Üì
       Works                    Shows 0 (empty tables)

    What I Found:
    - Dashboard called tenantService.getTotalTenants() ‚Üí counts Tenant entities (empty)
    - List pages called customerService.findTenants() ‚Üí counts Customer entities (has data)

    How It Should Work:
    - Customer table: Primary storage for all customer types (property owners, tenants, contractors)
    - Specialized tables: Either deprecated or used for specific extended attributes
    - All counts and lists: Should use Customer entity methods consistently

    3. Authorization Filtering Inconsistencies

    The Problem:
    Authorization filtering was applied inconsistently across the system:

    - Some pages filtered by user permissions
    - Others showed global counts
    - Same entities had different access patterns in different controllers

    Pattern I Established:
    - Individual list pages: Filter by user's assigned customers (unless MANAGER/SUPER_ADMIN)
    - Dashboard statistics: Show global counts (system-wide overview)
    - Admin pages: Always show global data
