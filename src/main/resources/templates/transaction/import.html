Import Guide
Supported Formats:
  • CSV: Spreadsheet format with headers
  • JSON: Structured data format
  • JSON Paste: Direct copy/paste from APIs
Required Fields:
  • • transaction_date - Date in YYYY-MM-DD format
  • • amount - Decimal (negative for payments out)
Recommended Fields:
  • • lease_reference - Highly Recommended! Links to specific lease (e.g., LEASE-BH-F1-2025)
  • • category - rent, maintenance, commission (helps with auto-matching)
  • • property_reference - Property name/address (fuzzy matched)
  • • customer_reference - Customer name/email (fuzzy matched)
Other Optional Fields:
  • • description, transaction_type (auto-generated if blank)
  • • subcategory, bank_reference, payment_method, notes
Tip: Include lease_reference to automatically link rent payments to leases. Use format: LEASE-{CODE}-{UNIT}-{YEAR} (e.g., LEASE-BH-F1-2025)
Automatic Lease Matching:

If you don't provide lease_reference, the system will automatically try to match rent transactions to leases using property + customer + date. For best results, include category="rent", property_reference, and customer_reference.

Tip: Manual lease_reference is more reliable than auto-matching!

Advanced Fields (Click to expand)

Advanced Import Fields:

  • incoming_transaction_amount - For rent payments, triggers automatic commission/owner split
  • beneficiary_type - Balance tracking (agency, beneficiary, contractor, beneficiary_payment)
  • counterparty_name - Entity involved (for contractor payments, bank names)
  • payment_source - Link to payment source (OLD_ACCOUNT, BANK_STATEMENT, etc.)
  • payprop_property_id, payprop_tenant_id, payprop_transaction_id - PayProp enrichment

Note: Most users don't need these fields. Use them only if you understand their purpose.
